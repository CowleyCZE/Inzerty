## 🪝 Hook Development Prompts

### Prompt: Create Custom Hook with Complex State

```
Create a custom hook called [hookName] for [feature]:

Features:
- Manage complex state with multiple sub-states
- Fetch data from /api/[endpoint]
- Handle real-time updates via WebSocket
- Include error handling and retry logic
- Provide loading indicator

Implementation:
1. Define UseHookNameReturn interface with all return values
2. Implement hook with useState, useEffect, useCallback

## 🧪 Testing Prompts

### Prompt: Write Comprehensive Component Tests

```
Write tests for component [ComponentName]:

Test Scenarios:
1. Component Rendering
   - Renders without props
   - Renders with all props
   - Renders with optional props
   
2. User Interactions
   - Button clicks trigger callbacks
   - Form inputs update state
   - Keyboard navigation works (accessibility)
   
3. State Management
   - State updates correctly
   - Props changes trigger updates
   - Callbacks are called with correct data
   
4. Edge Cases
   - Null/undefined data handling
   - Empty lists/arrays
   - Loading states
   - Error states

Implementation:
- Use React Testing Library (user-focused tests)
- Mock external dependencies (hooks, services)
- Use vi.fn() for callbacks
- Create fixture data for tests
- Achieve 80%+ code coverage

Output:
- [ComponentName].test.tsx with 10+ tests
- Fixtures file if reused across tests
- All tests passing
```

### Prompt: Test Custom Hook

```
Write comprehensive tests for hook [hookName]:

Test Areas:
1. Initial State
   - Hook returns correct initial values
   - No side effects on mount
   
2. Side Effects
   - Data fetches on mount
   - Updates state correctly
   - Handles loading state
   
3. Refetch Logic
   - refetch() triggers new fetch
   - Handles success case
   - Handles error case
   
4. Edge Cases
   - Component unmounts during fetch
   - Multiple calls don't create race conditions
   - Error recovery works
   
5. Dependencies
   - Dependencies are correct
   - Hook updates when deps change

Implementation:
- Use Vitest for hook testing
- Mock axios or API calls
- Use act() for state changes
- Test with various data scenarios
- Verify no memory leaks with cleanup
```

## 🐛 Debugging & Bug Fix Prompts

### Prompt: Debug and Fix Issue

```
Debug and fix the issue: [Description]

Reproduction:
[Steps to reproduce, expected vs actual behavior]

Investigation Steps:
1. Locate the problematic code
2. Add logging to understand flow
3. Check related dependencies
4. Review recent changes
5. Identify root cause

Fix Requirements:
- Fix the actual issue (not just symptoms)
- Add error handling if missing
- Add regression test to prevent recurrence
- Test thoroughly before completing
- Document the fix in comments

Must verify:
- Existing tests still pass
- New test covers the bug scenario
- No new bugs introduced
- Code style maintained
- TypeScript types correct
```

### Prompt: Fix WebSocket Connection Issues

```
Fix WebSocket connection issue in [hook/component]:

Debugging Checklist:
1. Is WebSocket URL correct? (env variable)
2. Is backend WebSocket server running?
3. Is connection being established?
4. Are event listeners being attached?
5. Is cleanup happening in useEffect?
6. Are event handlers properly bound?
7. Is the connection state managed correctly?

Common Issues to Check:
- Missing useEffect cleanup (memory leak)
- Event listeners attached multiple times
- Connection not stored in ref
- Missing error handlers
- Reconnection logic not implemented

Fix Implementation:
- Ensure useEffect cleanup removes listeners
- Store WebSocket connection in useRef
- Implement reconnection with exponential backoff
- Add error event handlers
- Log connection state for debugging
- Test with backend stopped/restarted
```

```
Create API route for [resource]:

Endpoints Required:
- GET /api/[resource] - List all with pagination
- GET /api/[resource]/:id - Get single
- POST /api/[resource] - Create new
- PUT /api/[resource]/:id - Update
- DELETE /api/[resource]/:id - Delete

Implementation:
1. Create route file: /backend/src/routes/[resource].ts
2. Create controller: /backend/src/controllers/[resource]Controller.ts
3. Create service: /backend/src/services/[resource]Service.ts
4. Update database queries if needed

Each endpoint must:
- Validate input (400 if invalid)
- Handle errors (500 for server errors)
- Return standard JSON: { success: boolean, data?: T, message?: string }

- Log errors with context

Testing:
- Create API tests
- Test all happy paths
- Test error cases (404, 400, 500)
- Test with invalid data
- Verify HTTP status codes
```

### Prompt: Add Database Migration

```
Create database migration for [changes]:

Changes:
[Describe what database changes needed]

Implementation:
1. Create migration file: /backend/src/database/migrations/[timestamp]_[name].ts
2. Implement up() function for applying changes
3. Implement down() function for reverting changes

- Ensure backward compatibility
