# 📋 Inzerty Development Instructions & Rules

## 🎯 Core Principles

1. **Type Safety First** - Every function must have proper TypeScript types
2. **Component Isolation** - Components should be reusable, well-documented with JSDoc
3. **Backend Scalability** - Services must be testable and independent
4. **Error Handling** - All async operations must have try-catch + logging
5. **Real-time Ready** - Always consider WebSocket side-effects

---

## 🏗️ Component Development Rules

### File Structure & Naming

```
components/
├── MyComponent.tsx          # Main component
├── MyComponent.test.tsx     # Unit tests
└── SubFolder/
    ├── SubComponent.tsx
    └── style.ts             # If complex styling needed
```

**Rules:**
- Component files: **PascalCase** (e.g., `MonitoringDashboard.tsx`)
- Hook files: **camelCase** with `use` prefix (e.g., `useConfig.ts`)
- Service files: **camelCase** with `Service` suffix (e.g., `geminiService.ts`)
- Utility files: **camelCase** or **under_score** for grouping

### Component Template

Every new component should follow this structure:


### Styling Rules

- **Framework**: Tailwind CSS only
- **Approach**: Inline `className` attributes
- **No CSS files** (unless absolutely necessary for complex animations)


## 🪝 Custom Hook Rules

### Hook Template

```typescript
import { useState, useEffect, useCallback } from 'react';

interface UseMyHookResult {
  state: string;
  loading: boolean;
  error: Error | null;
  refetch: () => Promise<void>;
}

/**
 * useMyHook - What this hook does
 * 
 * @param dependency - Explanation of parameter
 * @returns Object with state, loading, error, handlers
 * 
 * @example
 * const { state, loading, refetch } = useMyHook(config);
 */
export const useMyHook = (dependency: any): UseMyHookResult => {
  const [state, setState] = useState<string>('');
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState<Error | null>(null);

  const refetch = useCallback(async () => {
    setLoading(true);
    setError(null);
    try {
      // Fetch logic
    } catch (err) {
      setError(err instanceof Error ? err : new Error(String(err)));
    } finally {
      setLoading(false);
    }
  }, [dependency]);

  useEffect(() => {
    refetch();
  }, [dependency, refetch]);

  return { state, loading, error, refetch };
};
```

### Hook Rules

1. **Always return typed object** - Never return array destructuring
2. **Include loading & error states** - Standard pattern
3. **Add cleanup in useEffect** - Prevent memory leaks
4. **Document parameters** - Use JSDoc for clarity
5. **Memoize callbacks** - Use `useCallback` for event handlers

## 🔌 Service Layer Rules

### Service Template

```typescript
import axios from 'axios';
import { Ad, ScraperResult } from '../types';

/**
 * myService - Handles all API calls for feature X
 */
class MyService {
  private baseUrl = process.env.REACT_APP_API_URL || 'http://localhost:3001';

  /**
   * Fetch data from backend
   */
  async fetchData(id: string): Promise<Ad> {
    try {
      const response = await axios.get(`${this.baseUrl}/api/ads/${id}`);
      return response.data;
    } catch (error) {
      console.error('Error fetching data:', error);
      throw error;
    }
  }

  /**
   * Perform some action
   */
  async performAction(payload: any): Promise<void> {
    try {
      await axios.post(`${this.baseUrl}/api/action`, payload);
    } catch (error) {
      console.error('Error performing action:', error);
      throw error;
    }
  }
}

// Single instance (singleton pattern)
export default new MyService();
```

### Service Rules

1. **Singleton pattern** - Export single instance
2. **Error logging** - Every catch block must log to console
3. **Proper typing** - All parameters and returns typed
4. **Timeout handling** - Long operations need timeouts
5. **No state management** - Services are stateless

## 🗄️ Backend Rules

### Controller Template

```typescript
import { Request, Response } from 'express';
import { Ad } from '../types';

export const myController = {
  /**
   * GET /api/endpoint
   */
  async getAll(req: Request, res: Response): Promise<void> {
    try {
      const data = await myService.getAll();
      res.json({ success: true, data });
    } catch (error) {
      res.status(500).json({ 
        success: false, 
        message: error instanceof Error ? error.message : 'Unknown error' 
      });
    }
  },

  /**
   * POST /api/endpoint
   */
  async create(req: Request, res: Response): Promise<void> {
    try {
      const { name, description } = req.body;
      
      // Validation
      if (!name) {
        res.status(400).json({ success: false, message: 'Name is required' });
        return;
      }

      const result = await myService.create({ name, description });
      res.status(201).json({ success: true, data: result });
    } catch (error) {
      res.status(500).json({ 
        success: false, 
        message: error instanceof Error ? error.message : 'Unknown error' 
      });
    }
  }
};
```

### Route Template

```typescript
import { Router } from 'express';
import { myController } from '../controllers/myController';

const router = Router();

// GET /api/my-endpoint
router.get('/', myController.getAll);

// POST /api/my-endpoint
router.post('/', myController.create);

// GET /api/my-endpoint/:id
router.get('/:id', myController.getById);

// PUT /api/my-endpoint/:id
router.put('/:id', myController.update);

// DELETE /api/my-endpoint/:id
router.delete('/:id', myController.delete);

export default router;
```

### Backend Rules

1. **Every endpoint returns JSON** - `{ success: boolean, data?: any, message?: string }`
2. **HTTP status codes** - 200 (OK), 201 (Created), 400 (Bad Request), 404 (Not Found), 500 (Server Error)
3. **Input validation** - Always validate before processing
4. **Error logging** - Log errors with context (user, action, data)
5. **Async/await only** - No callback hell or promise chains

## 🧪 Testing Rules

### Test Template

```typescript
import { describe, it, expect, beforeEach, afterEach, vi } from 'vitest';
import { render, screen, fireEvent } from '@testing-library/react';
import MyComponent from './MyComponent';

describe('MyComponent', () => {
  beforeEach(() => {
    vi.clearAllMocks();
  });

  afterEach(() => {
    vi.restoreAllMocks();
  });

  it('should render with title', () => {
    render(<MyComponent title="Test Title" />);
    expect(screen.getByText('Test Title')).toBeInTheDocument();
  });

  it('should call onAction when button is clicked', () => {
    const mockHandler = vi.fn();
    render(<MyComponent title="Test" onAction={mockHandler} />);
    
    fireEvent.click(screen.getByRole('button'));
    expect(mockHandler).toHaveBeenCalled();
  });

  it('should handle loading state', async () => {
    render(<MyComponent title="Test" />);
    expect(screen.getByText('Loading...')).toBeInTheDocument();
  });
});
```

### Testing Rules

1. **One `describe` per component/hook** - Organize tests logically
2. **Descriptive test names** - `it('should...')`
3. **Arrange-Act-Assert pattern** - Clear test structure
4. **Mock external dependencies** - Use `vi.fn()`, `vi.mock()`
5. **Test user behavior** - Not implementation details
6. **Minimum coverage**: 80% for critical components

## 🔄 State Management Rules

### Rule 1: Use Custom Hooks for Feature State

```typescript
// ✅ CORRECT - Feature state in custom hook
const useMyFeature = () => {
  const [items, setItems] = useState([]);
  const [filter, setFilter] = useState('');

  const filtered = useMemo(() => 
    items.filter(item => item.includes(filter)),
    [items, filter]
  );

  return { items, setItems, filter, setFilter, filtered };
};

// ❌ WRONG - App-level state for everything
const [everythingState, setEverythingState] = useState({
  items: [],
  filter: '',
  // ... more stuff
});
```

### Rule 2: Separate Business Logic from UI

```typescript
// ✅ CORRECT - Logic in hook, UI in component
const MyComponent = () => {
  const { items, filtered, setFilter } = useMyFeature();

  return (
    <div>
      <input onChange={(e) => setFilter(e.target.value)} />
      {filtered.map(item => <div key={item}>{item}</div>)}
    </div>
  );
};

// ❌ WRONG - Mixed logic and UI
const MyComponent = () => {
  const [items, setItems] = useState([]);
  // ... 50 more lines of logic ...
  return <div>{/* JSX */}</div>;
};
```

## 🚨 Error Handling Rules

### Rule 1: Always Use Try-Catch for Async

```typescript
// ✅ CORRECT
const handleFetch = async () => {
  try {
    const data = await fetchData();
    setState(data);
  } catch (error) {
    const message = error instanceof Error ? error.message : 'Unknown error';
    console.error('Fetch failed:', message);
    setError(message);
  }
};

// ❌ WRONG
const handleFetch = async () => {
  const data = await fetchData(); // What if this fails?
  setState(data);
};
```

### Rule 2: Provide User Feedback

```typescript
// ✅ CORRECT - User knows what happened
if (!formData.email) {
  setError('Email is required');
  return;
}

// ❌ WRONG - Silent failure
if (!formData.email) {
  return; // User has no idea what went wrong
}
```

### Rule 3: Log with Context

```typescript
// ✅ CORRECT
console.error('Failed to save ad:', {
  adId: ad.id,
  error: error.message,
  timestamp: new Date().toISOString(),
});

// ❌ WRONG
console.error('Error'); // What error? Where? When?
```

## 🎨 Code Quality Standards

### ESLint/TypeScript Issues - Must Fix

- ❌ Any `any` type (use `unknown` if needed)
- ❌ Unused variables
- ❌ Missing `return` types on functions
- ❌ Unhandled Promise rejections
- ❌ Console.log left in production code

### Performance Rules

- Memoize expensive computations (`useMemo`)
- Memoize callback functions (`useCallback`)
- Use virtual scrolling for large lists (200+ items)
- Lazy load components with `React.lazy()`
- Split large components into smaller ones

### Accessibility Rules

- All interactive elements must have `aria-*` attributes
- Images must have `alt` text
- Buttons must have meaningful labels
- Color must not be the only indicator
- Form inputs must have associated labels

## ⚠️ Common Mistakes to Avoid

| Mistake | Solution |
|---------|----------|
| Not handling null/undefined | Use optional chaining `obj?.prop` |
| Forgetting dependency arrays | Always specify hook dependencies |
| Mutating state directly | Always use setState |
| Missing error boundaries | Wrap risky components |
| Not cleaning up effects | Return cleanup function in useEffect |
| Props drilling too deep | Use custom hooks or context |
| Hardcoding URLs | Use environment variables |
| Ignoring WebSocket events | Track in-flight requests |
